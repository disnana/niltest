# 多言語化とロケール拡張

niltest の利用者向け出力（テスト結果、比較エラー、生成される docstring）は日本語と英語に対応しています。言語を明示しない場合は Windows、macOS、Linux の OS 言語設定を使い、取得できない・未対応の場合は英語になります。

```python
import niltest

niltest.configure(language="ja")
niltest.configure(language="en")
```

環境変数でも指定できます。

```bash
NILTEST_LANGUAGE=en python app.py
```

## 独自ロケールを追加する

アプリケーションやプラグインは `register_locale()` でメッセージカタログを登録できます。翻訳漏れを防ぐため、英語カタログと同じすべてのキーが必要です。

```python
from niltest import configure, register_locale

register_locale("fr", {
    "scenario": "Scénario",
    "description": "Description",
    "returns": "Retour",
    "type_check_only": "vérification de type uniquement",
    "validator": "validateur",
    "no_cases": "aucun cas n'a pu être collecté.",
    "result": "Résultat : {passed} réussis / {failed} échoués",
    "value_mismatch": "Valeur différente :\\n  expected={expected!r}\\n  actual  ={actual!r}",
    "type_mismatch": "Type différent : actual={actual}, expected={expected}",
    "fields_mismatch": "Champs différents :\\n  expected={expected}\\n  actual  ={actual}",
    "validator_false": "Le validateur a renvoyé False\\n  actual={actual!r}",
    "validator_error": "Erreur du validateur : {error}",
})
configure(language="fr")
```

`ja-JP` や `en_US` のような指定は、それぞれ `ja` と `en` として扱われます。登録済みロケールを置き換える場合だけ `overwrite=True` を指定してください。

## 組み込み言語とテンプレート

組み込みカタログは `niltest/locales/` に言語ごとのモジュールとしてあります。新しい組み込み言語を始めるときは、`niltest/locales/template.py` をコピーしてすべての `TODO` を翻訳してください。外部パッケージは内部フォルダを編集せず、`register_locale()` で登録します。
